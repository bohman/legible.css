Grids and maths

We start by setting some default values. Font size and line height will determine the size of the grid. Block spacing determines how many grid units should be between each block level element.

The math behind headers is tricky for mathtarded people like me. Once we have a baseline grid with values above, we need to calculate size and line-height of the different headers. In order to do that, we add two more options. One is scale, which allows you to control how much bigger the headers should be relative to body type. The most common typographic scale tells us that type should be incremented in steps - often exponentially. That lets us set font size multiplier (which we'll shorten xfs).

The formula per header looks like this:

  • $header-font-size = scalexfs * font size
  • $header-number-of-lines = $header-font-size / line height
  • $header-line-height = line height * $header-number-of-lines
